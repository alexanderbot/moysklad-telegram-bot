# Автоматическая рассылка статистики

## Обзор

Добавлен функционал автоматической отправки статистики пользователям по расписанию с возможностью управления уведомлениями.

## Расписание рассылок

Все уведомления отправляются по московскому времени (Europe/Moscow, UTC+3):

### 1. Ежедневная статистика
- **Время:** 9:00 утра
- **Период:** Статистика за вчера
- **Формат:** Объединенный отчет (розничные продажи + заказы покупателей)

### 2. Недельная статистика
- **Время:** Понедельник, 9:05 утра
- **Период:** Статистика за прошлую неделю (понедельник-воскресенье)
- **Формат:** Объединенный отчет

### 3. Месячная статистика
- **Время:** 1 число месяца, 9:00 утра
- **Период:** Отчет о продажах за прошлый месяц
- **Формат:** Объединенный отчет

## Управление уведомлениями

### Команда /notifications

Пользователи могут управлять получением автоматических отчетов через команду `/notifications`:

```
/notifications
```

### Интерфейс управления

1. **Просмотр статуса:**
   - Показывает текущий статус уведомлений (включены/выключены)
   - Отображает расписание рассылок

2. **Включение уведомлений:**
   - Кнопка "🔔 Включить уведомления"
   - Пользователь начнет получать автоматические отчеты

3. **Выключение уведомлений:**
   - Кнопка "🔕 Выключить уведомления"
   - Остановка автоматической рассылки

## Технические детали

### Новые файлы

#### 1. `scheduler.py`
Модуль планировщика с классом `StatisticsScheduler`:
- `__init__()` - Инициализация с timezone='Europe/Moscow'
- `start()` - Запуск планировщика с настройкой cron-задач
- `stop()` - Остановка планировщика
- `_send_daily_report()` - Отправка ежедневных отчетов
- `_send_weekly_report()` - Отправка недельных отчетов
- `_send_monthly_report()` - Отправка месячных отчетов
- `_send_reports_to_users()` - Общий метод отправки
- `_format_scheduled_report()` - Форматирование отчета

**Cron-выражения:**
```python
# Ежедневно в 9:00
CronTrigger(hour=9, minute=0, timezone='Europe/Moscow')

# Понедельник в 9:05
CronTrigger(day_of_week='mon', hour=9, minute=5, timezone='Europe/Moscow')

# 1 число месяца в 9:00
CronTrigger(day=1, hour=9, minute=0, timezone='Europe/Moscow')
```

## Логика работы

### Получение списка получателей

При наступлении запланированного времени:

1. Планировщик запускает соответствующую функцию отправки
2. Из БД получается список пользователей с:
   - `notification_enabled = 1`
   - `api_token_encrypted IS NOT NULL`
   - `is_active = 1`

3. Для каждого пользователя:
   - Расшифровывается API-токен
   - Создается экземпляр `MoyskladAPI`
   - Запрашивается `get_combined_sales_report()` за нужный период
   - Форматируется и отправляется отчет
   - Логируется в `request_logs` с типом `scheduled_*`

### Обработка ошибок

- **Невалидный токен:** Пропуск пользователя, запись в лог
- **API недоступен:** Ошибка логируется, продолжение со следующим пользователем
- **Нет данных:** Отправка уведомления об отсутствии данных
- **Бот заблокирован:** Обработка исключения Telegram API

## Тестирование

### Ручное тестирование команды /notifications

1. Запустите бота: `python main.py`
2. Отправьте `/notifications` в чат с ботом
3. Проверьте отображение статуса
4. Нажмите "Включить уведомления"
5. Убедитесь, что статус изменился на "Включены ✅"
6. Нажмите "Выключить уведомления"
7. Проверьте изменение статуса на "Выключены ❌"

### Тестирование планировщика

Для быстрого тестирования можно временно изменить время в `scheduler.py`:

```python
# Вместо
CronTrigger(hour=9, minute=0, timezone='Europe/Moscow')

# Использовать (например, через 2 минуты)
CronTrigger(hour=14, minute=32, timezone='Europe/Moscow')
```

Или добавить тестовую задачу, которая запускается каждую минуту:

```python
self.scheduler.add_job(
    self._send_daily_report,
    CronTrigger(minute='*', timezone='Europe/Moscow'),
    id='test_stats',
    name='Тестовая статистика',
    replace_existing=True
)
```

### Проверка базы данных

Убедитесь, что настройки сохраняются:

```python
# В Python консоли
from database import Database
db = Database('data/bot_database.db')

# Проверить пользователей с уведомлениями
users = db.get_users_with_notifications()

# Проверить статус конкретного пользователя
status = db.get_notification_status(123456789)  # telegram_id
print(f"Статус уведомлений: {status}")
```

## Безопасность

- API-токены хранятся в зашифрованном виде
- Расшифровка происходит только на момент отправки отчета
- Пользователь может отключить уведомления в любой момент
- Проверка активности пользователя (`is_active = 1`)
